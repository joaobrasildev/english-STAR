# ADR-001: Prioritize a question-first answer screen layout

## Status

Accepted

## Date

2026-05-25

## Context

The current answer screen makes it harder for the user to re-read the question while writing. The user wants the question to stay easier to reference, to preserve intentional question line breaks from the original input, and to move the answer fields closer to the question so the main writing area feels more natural on desktop and laptop screens. The primary success condition is reducing the need to scroll or visually jump away from the question during practice.

## Decision

The product will prioritize a question-first answer screen layout. The question will remain the primary reference point, with preserved line breaks except for unnecessary trailing blank lines. The STAR response fields will sit directly below the question in the main horizontal area, while the timer and combined answer preview will move to a lower-priority area beneath the main writing flow.

## Alternatives Considered

### Alternative 1: Collapse the experience into a single response block

- **Description**: Keep the question visible, but replace the current STAR-first interaction with a more unified answer-writing area.
- **Pros**: Simpler visual layout and less panel switching.
- **Cons**: Weakens the structured STAR guidance that the product currently uses to help the user answer clearly.
- **Why rejected**: The user wants better readability and layout, not a reduced emphasis on STAR structure.

### Alternative 2: Keep the current layout but make secondary panels collapsible

- **Description**: Preserve the current screen structure and reduce clutter by collapsing the timer or combined answer panel.
- **Pros**: Smaller product change and limited disruption to the current screen.
- **Cons**: Does not fully solve the main problem of keeping the question easy to re-read while writing.
- **Why rejected**: The user’s primary need is a stronger question-to-answer flow, not just less visible clutter.

## Consequences

### Positive

- The answer screen becomes easier to use during active writing.
- The question remains easier to reference without constant scrolling or visual switching.
- The product becomes more faithful to the formatting the user entered for the question.

### Negative

- The current visual hierarchy of timer and combined answer preview becomes less prominent.
- Some users who prefer the existing split emphasis may need a short adjustment period.

### Risks

- If the question area becomes too visually dominant, the writing area could feel crowded.
- If formatting is preserved too literally, some malformed input could still look messy.
- Desktop-first optimization could leave smaller screens needing a later pass for refinement.

## Implementation Notes

This decision applies to the product design for the answer screen only. It should not expand the scope into timer redesign, summary redesign, or broader navigation changes.
